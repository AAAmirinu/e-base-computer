# 連続値の長期保存とEリフレッシュ

## 1. 基本問題

Eセルは `0 <= d < e` の連続値を保持できる理想素子ですが、実用機では
温度、外乱、経年劣化、読み出し操作そのものによって値が少しずつ揺らぎます。

ここで重要なのは、連続値の長期保存は普通のデジタル保存より難しい、という点です。

デジタル値なら、少しずれても `0` か `1` かを判定して元へ戻せます。
しかし連続値では、ずれた値が「本来の値」なのか「劣化した値」なのかを、
単独のセルだけから判断できません。

そのため、E進コンピューターでは長期保存を次のように扱います。

```text
完全に動かない実数を保存する
```

ではなく、

```text
許容誤差つきの連続状態を、参照系と冗長性によって保守する
```

と考えます。

このほうが数学的にも工学的にも無理が少ないです。

## 2. DRAMリフレッシュとの違い

DRAMのリフレッシュは、弱くなった電荷を読み直して再書き込みする操作です。
値は基本的に二値なので、しきい値で判定できます。

Eセルのリフレッシュ、つまり **Eリフレッシュ** はもっと複雑です。

```text
温度を測る
揺らぎを見積もる
参照セルと比較する
許容誤差内か判定する
必要なら値を補正する
Eキャリーを再正規化する
ガードバンドを再設定する
分割数や精度メタデータを更新する
```

つまり、Eリフレッシュは単なる再書き込みではなく、
**校正、冷却、正規化、意味の確認** を含む保守サイクルです。

## 3. 参照セル

長期保存には、通常のEセルとは別に **参照セル** が必要になります。

参照セルは、保存したいデータそのものではなく、装置のずれを測るための
基準値を保持します。

例:

- `0`
- `1`
- `e / 2`
- 既知の温度応答を持つ試験値
- 互いの和が一定になる対セル

リフレッシュ時には、まず参照セルを測って、装置全体がどの方向へどの程度
ずれているかを見積もります。

これにより、保存データを直接疑う前に「測定器のほうがずれていないか」を
確認できます。

## 4. 冗長保存

連続値を1つのセルだけに保存すると、ずれを検出できません。
そのため重要な値は、複数のセルに冗長化して保存します。

代表的な方法:

- **複製保存**: 同じ値を複数セルへ入れ、中央値や平均で復元する。
- **差分保存**: `x` と `e - x` のような関係を持つ対で保存する。
- **和一定保存**: 複数セルの和が既知の値になるようにする。
- **係数分散保存**: 1つの値を複数の係数へ分散して保存する。
- **離散退避**: 重要な粗い部分だけ三値または二値メモリにも保存する。

この世界の高信頼Eメモリは、見かけ上は1つのEセルに見えても、内部では
複数の物理セルを使っているはずです。

## 5. Eスクラブ

通常のメモリでは、誤り訂正のためにメモリスクラブが行われることがあります。
E進機械にも **Eスクラブ** があります。

Eスクラブは、Eメモリ全体を巡回して次を行います。

- 温度と揺らぎを測る。
- 参照セルのずれを記録する。
- 冗長セル同士の不一致を見つける。
- Eキャリーを再正規化する。
- 境界に近い離散値を安全域へ戻す。
- 必要なら分割数を降格する。
- 健全性メタデータを更新する。

Eスクラブは、DRAMリフレッシュよりもファイルシステムの検査や
RAIDの再同期に近い性格も持ちます。

## 6. 連続値の再投影

Eリフレッシュで特徴的なのは、値を単に戻すのではなく、
**許された状態空間へ再投影する** ことです。

例えば、あるE場が「物理モデルの係数」として保存されているなら、
係数は好き勝手な値ではなく、次のような制約を持つかもしれません。

- 全体のエネルギーが一定範囲にある。
- 確率分布の和が1になる。
- 隣接セルとの差が急に大きくならない。
- E進ワードとして正規化済みである。

リフレッシュ時には、測定された値をそのまま採用せず、これらの制約を満たす
最も近い状態へ戻します。

```text
観測された状態
  -> 参照補正
  -> 冗長復元
  -> 制約面への再投影
  -> 正規化
  -> 再保存
```

この処理は、E進コンピューター独特の「記憶の手入れ」です。

## 7. 保存階層

長期保存の方法は、保存したいものの性質によって変わります。

```text
短期:
  Eセル上に直接保持する。
  高速だが揺らぎや観測に弱い。

中期:
  冗長Eメモリで保持し、Eリフレッシュを繰り返す。
  AI状態、物理モデル、航法場に向く。

長期:
  Eスナップショットとして保存する。
  許容誤差、温度条件、参照セル情報、正規化規則を含める。

永久記録:
  重要な情報を二値または三値の離散記録へ落とす。
  連続的な細部は失われるが、法的・歴史的記録に向く。
```

連続値を完全に永久保存するより、時間が経つほど離散的な記録へ落としていく
ほうが自然です。

## 8. 記憶の品質等級

Eメモリには、用途ごとの品質等級があるはずです。

- **作業Eメモリ**: 高速だが、短時間でリフレッシュが必要。
- **冷却Eメモリ**: 低温で安定し、27値以上の分割を維持しやすい。
- **保管Eメモリ**: 低速だが、参照セルと冗長性が多い。
- **聖域バンク**: 航法、医療、人格モデルなど、重要なE場だけを置く。
- **化石化ストレージ**: 連続値を離散近似へ固めた長期記録。

この分類は、社会制度にも関わります。
例えば「人格E場を作業Eメモリに置いたまま放置すること」は危険行為として
規制されるかもしれません。

## 9. リフレッシュ周期

Eリフレッシュの周期は固定ではなく、状態によって変わります。

周期を短くする条件:

- 温度が高い。
- 分割数が多い。
- 観測回数が多い。
- 周辺セルの活動が激しい。
- 重要度が高い。
- 冗長セル間の差が広がっている。

周期を長くできる条件:

- 低温で安定している。
- 3値など粗いモードで使っている。
- 参照セルが安定している。
- 値がガードバンド中央にある。
- 多少の劣化が許される。

したがって、E進OSは単に「何ミリ秒ごとに全セルを刷新する」のではなく、
E場ごとにリフレッシュ周期を動的に決めます。

## 10. 観測と保存のジレンマ

連続値保存で面白いのは、健全性検査のために観測すると、その観測自体が
状態を少し荒らす可能性があることです。

このため、E進OSは次の判断を迫られます。

- 今すぐ観測して劣化を確認する。
- 観測せず、冷却と参照セルから状態を推定する。
- 重要領域だけ観測し、残りは推定で済ませる。
- 観測前にコピーを作る。
- 低分割モードへ降格してから観測する。

「見れば正確になる」とは限らないのが、E進記憶のよい癖です。
作中では「今は見ないほうが保存状態がよい」という判断が成立します。

## 11. 劣化の扱い

E進記憶では、劣化は単なるビット化けではありません。

劣化の種類:

- **漂移**: 連続値が少しずつずれる。
- **拡散**: 近い状態の区別が曖昧になる。
- **境界接近**: 離散モードのしきい値に近づく。
- **相関崩れ**: 冗長セル同士の関係が崩れる。
- **正規化乱れ**: Eキャリーの整合性が悪くなる。
- **意味劣化**: 連続値としては残るが、元の解釈モードでは使えなくなる。

これらを検出し、どこまで許容するかを決めるのがE進ファイルシステムや
E進カーネルの役割です。

## 12. 例外と警告

長期保存に関する例外は、次のようになります。

- **リフレッシュ遅延**: 予定されたEリフレッシュが間に合わない。
- **参照ずれ**: 参照セル群が想定より大きくずれている。
- **冗長不一致**: 複製されたEセルの値が許容範囲を超えて違う。
- **再投影失敗**: 制約を満たす近い状態へ戻せない。
- **保存降格**: 連続値保存を諦め、低分割または離散記録へ落とす。
- **観測保留**: 観測による劣化が大きいため、検査が延期された。

これらはシステムログにも物語上の台詞にも使えます。

## 13. 作中での言い回し

技術者は次のように話すかもしれません。

- 「このE場、最後のリフレッシュから長すぎる」
- 「参照セルが泳いでいる。値じゃなくて測定系がずれてる」
- 「人格場を低温バンクへ移せ。作業Eメモリでは保たない」
- 「連続記憶は諦める。三値の骨格だけ残せ」
- 「見るな。観測した瞬間に保存精度が落ちる」
- 「これは壊れたんじゃない。再投影できない形に歪んだんだ」

E進コンピューターにおける長期保存は、棚にしまうというより、
庭を手入れし続けることに近いです。
