# E進コンピューター発展史

## 1. 基本方針

E進コンピューターの歴史は、普通のデジタルコンピューター史とは違います。

この世界では、安定した二値・三値デジタル制御が成立する前に、
`0 <= d < e` の連続値を保持するEセルが先に発見された、と考えると
かなり面白くなります。

つまり初期のE進機械は、CPUもOSも持たない **純アナログ計算装置** です。
人間が温度、配線、参照セル、入力曲線を調律し、機械は連続状態のまま
計算しました。

成熟後のE進コンピューターはEPUやOSに管理されますが、その前史には
かなり長い「E進アナログ時代」があります。

## 2. 時代区分

大まかな流れは次のようになります。

```text
理論前史
  -> E効果の発見
  -> 単セル計測器
  -> 純アナログE計算機
  -> Eキャリーと正規化器
  -> 参照セルとEリフレッシュ
  -> 三値制御機の登場
  -> 混成E進コンピューター
  -> 成熟したEPU/OS時代
```

この流れにすると、E進技術は「デジタル計算機の上に突然載った特殊アクセラレータ」
ではなく、独自の文化と事故史を持つ古い技術になります。

## 3. 理論前史

Eセルの前には、対数、指数関数、連続量の計算に関する数学と工学が発展します。

この時代の研究者は、まだEセルを持っていません。
しかし、成長、減衰、振動、流体、熱、人口、薬物濃度のような問題を扱う中で、
`e` を基準にした表現が自然に現れることを知っています。

この段階での道具:

- 対数表。
- 機械式積分器。
- 連続曲線をなぞる解析機。
- 熱や流体を使った模型計算。
- 指数関数的な減衰を利用する計測器。

この世界の学者にとって、`e` は単なる数学定数ではなく、
自然界の変化が好んで使う目盛りとして認識されていました。

## 4. E効果の発見

次に、ある物理素子が特定条件下で `0 <= d < e` の安定した連続状態を
持つことが発見されます。

作中ではこれを **E効果** と呼べます。

初期の理解では、Eセルは計算機部品ではなく、奇妙な計測素子です。

- 温度に敏感。
- 外乱に弱い。
- しかし連続値の保持が異常に滑らか。
- ある範囲を超えると自然に別の安定状態へ移る。
- `e` に近い境界で独特の折り返し挙動を見せる。

この時代の研究者は、Eセルを「数を入れる箱」としてではなく、
「自然対数的な目盛りを持つ物理現象」として扱います。

## 5. 単セル計測器の時代

最初に作られる実用品は、計算機ではなく計測器です。

Eセルは、広いダイナミックレンジを持つ連続量を圧縮して保持するのに向きます。
そのため、初期用途は次のようなものになります。

- 温度や圧力の精密記録。
- 薬物濃度や反応速度の記録。
- 天文観測の微弱信号保持。
- 航海や測量での補助計算。
- 減衰曲線の記録。

この時代のE機器は、まだプログラムできません。
職人が参照セルを合わせ、目盛りを読み、手で表へ写します。

作中用語:

- **E計器**
- **対数瓶**
- **保持管**
- **E目盛り**

## 6. 純アナログE計算機の時代

やがて複数のEセルを組み合わせ、加算器、乗算器、シフタ、正規化器の原型が
作られます。

ここで重要なのは、この時代にはまだ信頼できるデジタル制御機がないことです。
したがって、E進計算機は単体のアナログ機械として運用されます。

特徴:

- プログラムは配線盤、歯車、流体弁、光学マスクで組む。
- 条件分岐は弱く、ほとんどの計算は連続的な流れとして設計される。
- 入力はつまみ、曲線板、観測信号、参照セルで与える。
- 出力は針、記録紙、発光パターン、安定したEセル列として読む。
- 起動ではなく「調律」してから計算する。

この時代のE進機械は、現代的な意味での汎用コンピューターではありません。
むしろ、問題ごとに作られた巨大なアナログ計算炉です。

用途:

- 潮汐予測。
- 砲術表や軌道表の生成。
- 流体模型。
- 反応炉の安定解析。
- 初期の気象予測。
- 医療用の薬物動態モデル。

## 7. なぜデジタル制御なしで動けたのか

初期E進機械がデジタル制御なしで動けた理由は、そもそもアナログ計算が
命令列を逐次実行する方式ではないからです。

アナログ計算では、問題を回路や物理配置として作り、あとは系が自然に
状態を変化させます。

```text
命令を読む
  -> 計算する
  -> 次の命令へ進む
```

ではなく、

```text
物理系を組む
  -> 入力を与える
  -> 安定状態または時間発展を観測する
```

という形です。

そのため、初期E進機械に必要だったのはCPUではなく、次のようなものです。

- 安定した参照セル。
- 手動または機械式の切替器。
- 温度調律。
- 正規化のためのアナログ帰還回路。
- 出力を読む熟練技術者。

この時代の「プログラマ」は、コードを書く人ではなく、
E場の形を設計して調律する人でした。

## 8. 初期E進機械の弱点

純アナログE計算機は強力ですが、扱いづらい機械でもあります。

弱点:

- 再現性が低い。
- 計算条件を記録しにくい。
- 分岐や例外処理が苦手。
- 長時間運用で値が漂移する。
- 観測すると状態が乱れることがある。
- 問題を変えるたびに配線や温度条件を組み替える必要がある。
- 計算結果を他の装置へ正確に渡しにくい。

この弱点が、後のデジタル制御機を必要とする大きな理由になります。

## 9. Eキャリーと正規化器の発明

初期E機械を計算機らしくした大きな発明が **Eキャリー** と
**正規化器** です。

それ以前のEセル列は、連続値を並べた物理状態に近いものでした。
正規化器の発明により、Eセル列はE進ワードとして扱えるようになります。

この時代に生まれる部品:

- 桁上げ弁。
- E境界検出器。
- 対数シフタ。
- 正規化環。
- 畳み込み盤。

これにより、E進機械は単なる連続物理模型から、再利用可能な計算単位へ
近づきます。

ただし、まだ制御はほぼアナログです。
正規化器も、自動で命令を実行するのではなく、値が境界を越えたら物理的に
上位セルへ流すような帰還装置として作られます。

## 10. 参照セルと保守技術の時代

計算が大規模になると、保存値の漂移が問題になります。

ここで登場するのが、参照セル、冗長保存、Eリフレッシュ、Eスクラブです。

この時代に、E進機械は一気に「運用技術」を持ち始めます。

- 参照セルで装置のずれを測る。
- 冗長Eセルで重要値を守る。
- 定期的にEリフレッシュする。
- 温度日誌をつける。
- 調律師が機械の癖を読む。

初期のE進施設では、計算技術者と保守技術者の境界が曖昧です。
計算を成功させるには、数式だけでなく、装置の体調を読む必要があります。

## 11. 三値制御機の登場

次の転換点は、安定した三値素子の実用化です。

三値素子は、Eセルより安く、離散状態を安定して扱えます。
これにより、E進機械の周囲に制御用の三値計算機が付き始めます。

この時代の変化:

- 温度管理が自動化される。
- Eリフレッシュ周期を機械が決める。
- 分岐、停止、再試行ができるようになる。
- 計算条件を離散ログとして保存できる。
- Eセルのモード切替が安定する。
- E進機械が複数ジョブを扱えるようになる。

ここで初めて、E進機械は「計算炉」から「コンピューター」に近づきます。

ただし主役はまだE進アナログ部です。
三値制御機は、E炉の世話役として使われます。

## 12. 二値デジタル技術との合流

通信、暗号、事務処理、標準化の必要から、二値デジタル技術も発展します。

二値技術はE進計算に比べると地味ですが、次に強いです。

- 完全一致。
- 複製。
- 通信。
- 暗号。
- 大量生産。
- 互換性。

やがてE進機械は、二値・三値の制御系に囲まれた特殊演算装置になります。

この時代に、現在の資料でいうEPU、Eメモリ、E場、観測権限、Eスナップショットが
整備されていきます。

## 13. EPU時代

デジタル制御が成熟すると、E進アナログ部は **EPU** として標準化されます。

以前は施設ごとに癖があったE炉が、命令セットを持つ部品になります。

この時代の特徴:

- `ENORM`, `ESHIFT`, `EQUANT`, `EREFRESH` などの命令が標準化される。
- E場をOSが資源として管理する。
- 観測権限がセキュリティ機構に組み込まれる。
- Eスナップショットの形式が作られる。
- コンパイラがEPUと通常CPUへ処理を振り分ける。

ここでようやく、成熟したE進コンピューター像に到達します。

## 14. 旧式機の残存

成熟した時代にも、古い純アナログE機械は完全には消えません。

理由:

- 特定用途では非常に速い。
- デジタル制御を挟まないので観測遅延が少ない。
- 古い施設や軍事設備に残る。
- 現代EPUとは違う癖を持つため、特殊な計算に使える。
- 宗教的、文化的、職人的な価値を持つ。

旧式E炉を扱える調律師は少なくなり、半ば伝統技能になります。

作中では、最新のEPUでは再現できない古い航法場や、旧式E炉にしか読めない
連続記憶が出てくると面白いです。

## 15. 技術史上の対立

E進技術の歴史には、いくつかの対立がありそうです。

### アナログ派とデジタル派

アナログ派は、E場を直接扱うことを重視します。
デジタル派は、再現性、記録、標準化、安全性を重視します。

両者の妥協として、成熟期の混成アーキテクチャが生まれます。

### 観測派と保存派

観測派は、状態を読んで記録することで信頼性を上げようとします。
保存派は、読むこと自体が状態を荒らすと考え、なるべく観測しない運用を
好みます。

この対立は、EリフレッシュやEスナップショットの設計思想に影響します。

### 三値派と二値派

三値派は、Eセルとの相性や平衡三進法の美しさを主張します。
二値派は、通信、暗号、互換性、製造容易性を武器にします。

結果として、内部制御は三値、外部通信と記録は二値、連続計算はE進という
棲み分けになります。

## 16. 時代ごとの機械の見た目

### 初期E計器

研究室の机に置かれた、温度槽付きの精密計測器。
値は針や発光で読まれ、人間が記録します。

### 純アナログE計算炉

部屋全体を占める配線盤、冷却槽、参照セル列、記録紙。
計算前には数時間から数日かけて調律します。

### 三値制御付きE炉

前面に三値制御盤を持ち、温度、リフレッシュ、正規化の一部を自動化します。
まだ専門施設向けです。

### 混成E進コンピューター

通常CPU、三値制御部、EPU、Eメモリを持つ高級計算機。
OSがE場を管理します。

### 成熟E進基盤

宇宙船、都市インフラ、医療AI、気象制御に組み込まれ、一般人は内部がE進で
あることを意識しません。

## 17. 作中で使える歴史的事件

設定に厚みを出すなら、次のような事件を置けます。

- 初めてE効果が観測された実験。
- 純アナログE炉が人間の計算表を上回った日。
- 正規化器の暴走により、航法計算が全桁でずれた事故。
- 参照セルの重要性が認識された保存事故。
- 三値制御機がE炉の調律を自動化した転換点。
- 観測権限をめぐる最初の裁判。
- 古いE炉の連続記憶を現代EPUへ移植する難事業。

## 18. 設定としての結論

E進コンピューターは、最初から現代的なコンピューターだったわけではありません。

最初は、連続値を保持する不思議な計測素子でした。
それが巨大なアナログ計算炉になり、参照セルと正規化器を獲得し、
三値制御機に世話され、最後にデジタルOSから管理されるEPUへ収束しました。

この歴史にすると、成熟期のE進コンピューターにも古いアナログ文化の名残が
残ります。

- 起動ではなく「調律」。
- 読み出しではなく「観測」。
- 保存ではなく「保守」。
- バグではなく「意味のずれ」。
- 初期化ではなく「安定化」。

E進コンピューターは、デジタル計算機から派生した装置ではなく、
アナログ計算機として先に生まれ、後からデジタル文明に取り込まれた技術、
と考えるのがいちばん雰囲気が出ます。
