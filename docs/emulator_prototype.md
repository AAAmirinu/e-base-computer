# EPUエミュレータープロトタイプ実装メモ

## 1. 目的

このメモは、`docs/epu_spec.md` のうち、最初のプロトタイプで実装する範囲を
固定するためのものです。

プロトタイプの目的は、完成したOSやコンパイラを作ることではありません。
まず、EPUが命令列を受け取り、Eレジスタ、Eメモリ、温度、量子化、観測を
伴って実際に動くことを確認します。

## 2. 実装する範囲

プロトタイプでは次を実装します。

- Eレジスタ `ER0..ER15`
- Eポインタレジスタ `EP0..EP7`
- 状態フラグ `SR`
- 制御フラグ `CR`
- Eセル、E場、Eポインタ
- 簡易Eメモリバンク
- アセンブリ風命令のパーサ
- E進算術
- Eメモリ確保、保存、読み出し
- 量子化と逆量子化
- 簡易熱モデル
- Eリフレッシュ
- スナップショット
- 診断出力
- 構造化イベントログ
- 実行前後スナップショット

## 4. まだ実装しない範囲

次は、設定としては存在しますが、最初のプロトタイプでは簡略化します。

- 実OSのプロセス分離
- 暗号的な観測ログ
- 物理的な観測破壊の厳密な再現
- Eセル間の局所干渉
- 高度な指数基底変換
- 本格的な高級言語コンパイラ

## 5. 熱モデル

温度は簡易的に扱います。

命令を実行すると対象の温度が少し上がり、tickごとにバンクの冷却率に応じて
少し下がります。温度が高いほど安全に使える分割数が下がります。

プロトタイプでは、許可分割数を次の階段へ丸めます。

```text
3, 9, 27, 81, 243
```

要求分割数が温度上危険な場合、`allow_degrade` が有効なら降格し、
無効なら例外を出します。

## 6. 実行例

```text
ECONST ER0, 12.5
ECONST ER1, 4.25
EMUL ER2, ER0, ER1
ENORM ER2
ESHIFT ER3, ER2, 1
EOBS OUT0, ER3 ; precision=8
```

このプログラムは、`12.5 * 4.25 * e` をE進ワードとして計算し、
通常数値として観測します。

## 7. 可視化フック

プロトタイプEPUは、各命令の実行時に構造化イベントログを残します。

```text
epu.timeline()
```

で、命令名、ターゲット、状態フラグ、実行前スナップショット、実行後
スナップショットを取得できます。

これにより、後から次のような可視化を作れます。

- 命令タイムライン。
- Eレジスタの桁ラダー。
- Eメモリ上のE場マップ。
- 温度と許可分割数の推移。
- 観測によって `OBSERVATION_DIRTY` が立つ瞬間。
