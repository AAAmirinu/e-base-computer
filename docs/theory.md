# 架空E進コンピューターの基礎理論

## 1. 呼び名について

作中世界では、この機械を **E進コンピューター** または
**E-adic computer** と呼ぶことにします。理由は、ネイピア数 `e` を
基数のように扱い、`e^k` の係数列として値を保持するためです。

ただし、数学的に厳密な意味では、これは通常の **p進数** ではありません。
p進数は素数 `p` を基礎にする体系であり、`e` は整数でも素数でもなく、
超越数です。

したがって、この設定での「E進」は、通常のp進数をそのまま一般化した
ものではなく、**実数基数の位取り表現に、連続値の桁を組み合わせた
架空計算体系** です。作中用語としてはE進と呼びつつ、数学的には
この区別を残しておくと安全です。

## 2. 理想素子

基本素子は、ひとつの物理自由度に次の値を正確に保持できると仮定します。

```text
0 <= d < e
```

この資料では、この素子、またはそこに保持される値を **edit**、
**E桁**、または **Eセル** と呼びます。

現実の物理では、このような完全な連続値記憶は不可能に近いものです。
しかしSF設定として、特殊な物質、人工的に設計された物理法則、魔法的な
安定化場などによって存在すると仮定します。

重要なのは、素子の存在は架空でも、素子を使った演算規則は通常の数学で
説明できるようにすることです。

## 3. E進ワード

E進ワードは、整数の指数 `k` からE桁 `d_k` への有限写像です。

```text
W = { k -> d_k }
```

これは次の実数値を表します。

```text
value(W) = sum(d_k * e^k)
```

例:

```text
2.5 * e^3 + 0.25 * e^-1
```

正の指数は大きなスケールを、負の指数は細かな分解能を表します。
つまりE進ワードは、`e` を基準にした対数的な物差しの上で値を保持します。

## 4. Eキャリーと正規化

演算の途中では、桁の係数が `[0, e)` の範囲をはみ出すことがあります。
そのとき、通常の位取り記数法のように上位桁へ桁上げします。

```text
carry = floor(coefficient / e)
digit = coefficient - carry * e
coefficient[k + 1] += carry
```

この変換は値を変えません。

```text
coefficient * e^k
= digit * e^k + carry * e * e^k
= digit * e^k + carry * e^(k + 1)
```

キャリーは整数ですが、桁そのものは連続値でよい、という点がこの設定の
面白いところです。素子は架空でも、桁上げの代数は普通の実数計算です。

## 5. 加算

加算は、同じ指数の係数を足してから正規化します。

```text
C[k] = A[k] + B[k]
normalize(C)
```

正規化は値を保存するため、結果は次を表します。

```text
value(A) + value(B)
```

## 6. 乗算

乗算は畳み込みで表せます。

```text
C[i + j] += A[i] * B[j]
normalize(C)
```

理由は次の通りです。

```text
(a_i * e^i) * (b_j * e^j) = (a_i * b_j) * e^(i + j)
```

このあとEキャリーで正規化すれば、結果は次を表します。

```text
value(A) * value(B)
```

## 7. 符号と有限性

現在のシミュレータでは、有限長のE進ワードに外部の符号ビットを付けて
負数を表しています。

これは、負の数を無理に有限桁のE進表現だけで表そうとして、存在しない
一意性を主張してしまうのを避けるためです。

より濃いSF設定にするなら、半無限長ワードや、二の補数に似たE進補数表現を
導入できます。ただし、基礎モデルでは扱いません。

## 8. 数学的に破綻しない理由

この機械は、ありえないほど正確な素子を仮定します。
しかし、演算規則そのものは次だけで成り立っています。

- 実数
- 有限和
- `e` のべき乗
- `floor` を使った通常の桁上げ
- 畳み込みによる乗算

現実の物理がこの素子を許すとは言えません。
しかし、物語が素子の存在を認めるなら、計算規則は通常の数学の中で
一貫して扱えます。

## 9. 架空アーキテクチャの部品

作中世界では、次のような部品名を使えます。

- **Eセル**: `[0, e)` の値をひとつ保持する基本素子。
- **Eレジスタ**: 複数のEセルと符号情報を並べた記憶単位。
- **正規化器**: Eキャリーを伝播させ、桁を `[0, e)` に戻す回路。
- **対数シフタ**: 指数をずらし、値を `e^n` 倍する回路。
- **畳み込み器**: E進ワード同士の乗算を行う回路。
- **観測器**: E進ワードを人間向けの通常の実数表示に変換する装置。

この機械の奇妙さは、「1桁」がビットのような二値ではなく、
`0` から `e` までの連続的な窓になっている点にあります。
