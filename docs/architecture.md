# E進コンピューターの世界内アーキテクチャ

## 1. 前提

この世界では、少なくとも次の2種類の素子が実用化されているとします。

- **三値素子**: `0, 1, 2` の3状態を安定して保持できる安価な素子。
- **Eセル**: `0 <= d < e` の連続値を高精度に保持できる高価な素子。

この前提なら、社会全体がEセルだけで構成された万能計算機に移行するとは
考えにくいです。むしろ、用途ごとに次のような棲み分けが起きるほうが
自然です。

## 2. 採用される方式

### 通常の二進法

二進法は、既存のデジタル技術、通信、誤り訂正、暗号、制御ソフトウェアと
相性がよい方式です。

この世界でも、二進法は消えないはずです。理由は単純で、二値の論理は
判定が強く、ノイズに強く、設計資産が膨大だからです。

用途:

- 互換性が重要な汎用計算
- 通信プロトコル
- 暗号
- OSや制御系の中核
- 安価な大量生産機器

### 三値素子による三進法

三値素子が安定かつ安価なら、三進法はかなり魅力的です。

三値は `0, 1, 2` でもよいですが、論理設計では `-1, 0, +1` の
**平衡三進法** が特に便利です。符号つきの値を自然に扱えるためです。

用途:

- 省配線・高密度なデジタル回路
- 低消費電力な組み込み計算
- しきい値判定、状態機械
- Eセル周辺の制御回路

この世界では、三値素子は「高級なEセルを制御するための現実的なデジタル
基盤」になりそうです。

### Eセルに離散値を格納する方式

Eセルには連続値を保持できるので、もちろん離散値も格納できます。

例えば `[0, e)` を27区間に分ければ、1つのEセルに27状態を格納できます。
これは三値で言えば `3^3 = 27` なので、1セルに三値3桁分を詰め込む
イメージです。

ただし、この方式には問題があります。

- Eセルは高価で、単純な三値記憶に使うにはもったいない。
- 27分割すると、各状態の間隔は `e / 27` になり、ノイズ余裕が小さい。
- 論理演算のたびに、連続値を離散状態へ丸める量子化器が必要になる。
- 誤差が蓄積すると、隣の状態へ化ける危険がある。

そのため、Eセルを大量の離散メモリとして使うのは、通常は贅沢で不安定です。

しかし、強引だからこそSFとして面白い使い道があります。Eセルを
**アナログ演算器として使っている途中だけ、同じ回路を離散モードへ切り替える**
という設計です。

この場合、Eセルは「高価な記憶素子」ではなく、次のような
**可変意味の計算媒体** になります。

- 連続モード: 微分方程式、最適化、物理シミュレーションを解く。
- 三値モード: 制御分岐、条件判定、離散状態遷移を行う。
- 多値モード: 係数、タグ、短い命令列、近似された確率分布を保持する。

現実のコンピューターでも、純粋で美しい方式だけが勝つわけではありません。
キャッシュ、投機実行、ベクトル命令、GPU、JIT、互換レイヤーのように、
かなり複雑な仕組みを組み合わせて「結果として使いやすい機械」を作ります。

したがって、Eセルの離散利用も、設定として十分ありです。
ただし、主役にするなら「高価だが、連続計算と離散制御を同じ物理場で
切り替えられる」という利点を前面に出すと説得力が出ます。

## 3. もっとも自然な棲み分け

この世界で一般的に選ばれる構成は、次の混成型だと思われます。

```text
安価な二値回路:
  外部通信、既存資産、堅牢な制御、暗号

安価な三値回路:
  内部制御、状態機械、Eセルの読み書き制御、圧縮されたデジタル演算

高価なEセル回路:
  連続値計算、対数スケール演算、物理モデル、適応制御、特殊な多値記憶
```

つまり、E進コンピューターは「全てをEセルで置き換えたコンピューター」
ではなく、**Eセルを計算の心臓部に持つ混成計算機** として普及するのが
自然です。

## 4. 論理演算はどう実装されるか

Eセルそのものはアナログ素子なので、素のままではANDやORのような
論理演算を持ちません。論理演算には、値を安定した離散状態に写す
**判定器** が必要です。

三値論理なら、Eセルの値 `x` を次のように分類します。

```text
0 <= x < T0      -> 0
T0 <= x < T1     -> 1
T1 <= x < e      -> 2
```

ここで `T0`, `T1` はしきい値です。27値なら、しきい値を26本置きます。

論理演算には2段階あります。

```text
連続値の読み取り
  -> しきい値判定で離散記号へ変換
  -> 三値論理回路または多値論理表で演算
  -> 結果の記号をEセル内の代表値へ書き戻す
```

例えば三値ANDに相当する演算を `min(a, b)` と定義するなら、
三値記号上で `min` を取り、結果を `0`, `1`, `2` の代表値として
Eセルへ戻します。

ただし、Eセルの値が状態境界に近いと誤判定しやすくなります。
そのため実用機では、各状態に幅を持たせ、境界付近を使わない
**ガードバンド** を置くはずです。

```text
状態0の安全域 | 未使用域 | 状態1の安全域 | 未使用域 | 状態2の安全域
```

このガードバンドにより、1つのEセルに詰め込める離散状態数は
理論上の限界より少なくなります。

## 5. 27分割はありか

27分割は、設定としてかなりよい落とし所です。

理由は、27が `3^3` なので、三値論理の世界と接続しやすいからです。
Eセル1つを「三値3桁のパック」と見なせます。

```text
状態番号 n = a0 + 3*a1 + 9*a2
a0, a1, a2 はそれぞれ 0, 1, 2
```

ただし、27値を直接論理演算するより、読み出し時に三値3桁へ展開してから
三値回路で処理するほうが自然です。

Eセル上で27状態を直接処理する方式は、次の用途に向きます。

- 小さな命令やタグを1セルに圧縮する。
- アナログ演算中のモード番号を保持する。
- 短い状態機械をEセル列の中に埋め込む。
- 多少の誤差を許すAI的・適応制御的な処理を行う。

逆に、銀行の残高、暗号鍵、法的記録のような絶対に化けてはいけない情報は、
三値素子か二値素子で保持し、Eセルには置かないほうが自然です。

## 6. E進で微分積分はシフトになるのか

「E進法では微分積分が単純なビットシフトで表現できる」という話は、
そのままでは少し危険です。

普通の多項式

```text
f(x) = sum(a_k * x^k)
```

を微分すると、

```text
f'(x) = sum(k * a_k * x^(k - 1))
```

となります。指数が1つ下がるのでシフトに似ていますが、同時に `k` を
掛ける必要があります。つまり、単純なシフトだけではありません。

一方で、指数関数は特別です。

```text
d/dx e^x = e^x
```

さらに、関数を次のような指数基底で展開する場合を考えます。

```text
f(t) = sum(a_k * e^(lambda_k * t))
```

このとき微分は、

```text
f'(t) = sum(lambda_k * a_k * e^(lambda_k * t))
```

となります。これは基底ごとの係数に `lambda_k` を掛けるだけです。
フーリエ変換やラプラス変換で、微分が周波数領域の乗算になるのと似ています。

E進ワードの「指数をずらすシフト」は、値を `e^n` 倍する操作です。

```text
shift +1: value -> e * value
shift -1: value -> value / e
```

したがって、E進ワードそのものに対するシフトは
**微分積分そのもの** ではなく、**指数スケールの倍率変更** です。

ただし、作中設定としては次のようにすると面白く、かつ数学的にも無理が少ないです。

この世界のE進コンピューターは、物理量を普通の時間軸ではなく
**対数時間** や **指数基底の係数空間** に変換して扱う。
その空間では、減衰、成長、積分核、スケール変換の多くが
シフトや係数倍として表せる。

つまり正確には、

```text
微分積分が常にシフトになる
```

ではなく、

```text
指数基底や対数スケールに変換した問題では、
微分積分の一部がシフトや単純な係数演算に落ちる
```

です。

この言い方なら、かなり使えます。

## 7. 作中での使われ方

E進コンピューターが得意とするのは、次のような領域です。

- 連続値を直接扱う制御。
- 成長、減衰、拡散、振動のような指数的現象。
- 対数スケールで広い範囲を扱う観測装置。
- 物理シミュレーションや軌道計算。
- AIの内部状態や確率分布の近似表現。
- 三値制御と連続演算を頻繁に切り替える特殊用途。

一方で、苦手または不向きなのは次です。

- 完全な再現性が必要な事務処理。
- 暗号鍵の保存。
- 大量の安価な記録媒体。
- 境界判定だらけの単純な論理処理。

そのため、この世界のE進コンピューターは、日用品の中心というより、
宇宙船の航法炉、気象制御、医療用生体場モデル、巨大インフラ制御、
高等AIの感覚野のような場所で使われると説得力があります。

## 8. 設定としての結論

もっとも自然な結論は次です。

E進素子は、三値素子の上位互換ではありません。
三値素子は安く堅牢な離散論理に向き、Eセルは高価だが連続値と
対数スケールの計算に向きます。

ただし、Eセルを離散モードに切り替えて使う設計は十分ありえます。
それは「もったいない使い方」ではありますが、同じ物理回路で連続演算と
離散制御を往復できるなら、複雑なシステム全体としてはむしろ合理的です。

この強引さは、現実のコンピューターにある強引さとよく似ています。
単体の理屈としては複雑でも、巨大な処理能力、補正回路、コンパイラ、
キャッシュ、抽象化層が組み合わさることで、使う側にはひとつの強力な
計算環境として見えるのです。
